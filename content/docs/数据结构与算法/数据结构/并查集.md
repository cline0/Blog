title: "并查集"
weight: 4
---

并查集（Disjoint Set Union, DSU）又被叫作 **不相交集合数据结构（disjoint-set data structure）**。

一个并查集由多个集合组成，并使用一个**代表（representative）**来标识每一个集合。代表是该集合的成员。代表的选择可以使用特定的规则来选取。

可以支持 3 种操作：

- `MAKE-SET(x)`：建立一个新的集合，它的唯一成员是 x.
- `UNION(x, y)`：包含 x 和 y 的两个集合合并成一个新的集合。并分别将原来的集合删除。
- `FIND-SET(x)`：返回一个指针，该指针指向包含 x 的唯一集合的代表。

使用链表来表示每一个集合，集合中每个元素还包括指向链表头结点（默认头结点为集合的代表）的指针。则：

- `MAKE-SET(x)` 操作就是创建一个只包含 `x` 的链表
- `FIND-SET(x)` 操作使用一个 `Map<x, 该元素对应的节点的指针>` 来定位到该元素的节点，然后返回该节点中的代表指针
- `UNION(x, y)` 操作是将 y 所在的链表拼接到 x 所在的链表从而实现合并。但是，必须更新所有 y 中的代表指针，从而导致了 O(n) 的时间复杂度。

有一种关于 `UNION` 启发式策略是永远将短的链表合并到长的链表中。
